# 動画の一部だけを復元する

動画全体を復元する必要はありません。復元が必要なシーンだけを選べば、
残りの部分はそのままコピーされます。処理はずっと速くなり、それ以外の
部分は元の品質が保たれます。出力はフル尺の動画のままです。

## 区間エディター（GUI）

<img width="822" alt="Segment editor" src="https://github.com/user-attachments/assets/c67939a9-37de-46ae-b722-20c6a0933df7" />

キュー内の動画のはさみボタンから開きます。

- 動画をプレビューし、フレーム単位で正確に範囲を選択します。何も
  選択しなければ動画全体を復元します。
- **復元プレビュー**は、処理を確定する前に、現在の復元設定で現在の
  フレームや短い再生結果を表示します。
- 区間を選択すると、書き出しには元動画のコーデックが使われます —
  メイン画面の**エンコード**のコーデック設定は適用されません。未選択の
  部分がそのままコピーされるためです。区間処理に使えない動画の場合は、
  エディターが事前に知らせます。

### モザイクスキャン

エディターはモザイクのあるシーンを自動で探せます:

- 全フレーム、または 0.25～2 秒間隔でスキャンします。スキャンは GPU 上で
  実行され、**RTX 5090 では約 2,000 FPS** に達します。
- スキャン後に信頼度スライダーを調整すると琥珀色の検出区間が更新され、
  ワンクリックで紫色の復元対象の選択へ追加できます。

検出モデルと信頼度はキュー内の動画ごとに記憶され、最終処理でも
使われるため、動画ごとに異なる設定を使えます。

### より良いマスクを提案する

検出が正しくないと思ったら、今後のモデルの改善に協力できます。該当する
フレームで一時停止し、**より良いマスクを提案**をクリックして、各モザイク
領域の輪郭を描いてください — 描き方はエディターが案内します。モザイクの
縁がぼやけて薄くなっている場合は、そのぼかし部分も含めてください。

送信すると、フレームとマスクは**匿名**でアップロードされます。データは
アップロード前にお使いのマシン上で暗号化され、付加される情報はアプリの
バージョン、検出モデル名、フレーム解像度のみです — ファイル名、
タイムスタンプなど、あなたを特定できるものは一切含まれません。

## CLI: `--segments`

エディターの CLI 版です。時刻は秒または `HH:MM:SS.s` 形式で指定します:

```bash
jasna --input input.mp4 --output output.mp4 --segments "10-25,01:10-01:30.5"
```

Jasna は指定した範囲を復元し、それ以外をすべてコピーします。最も近い
安全なカット位置の前後の短い区間は再エンコードされますが復元は
されません — これは動画を継ぎ目なくつなぎ直すために必要です。

区間処理で使える条件:

- 現時点では NVIDIA GPU のみ。
- 一度に 1 本の動画のみ（フォルダ、画像、ストリーミングは不可）。
- 固定フレームレートの H.264、HEVC、AV1 入力と、MP4、MOV、MKV 出力。
- 出力コーデックは常に入力コーデックと同じになります。
- `--retarget-high-fps` とは併用できません。

互換性のない入力は、処理開始前に分かりやすいエラーで拒否されます。

## 作業ディレクトリ

区間出力の組み立て中、Jasna は一時ファイルを出力動画の隣の隠しフォルダに
保存します。別の（より速い、または大きい）ドライブに置きたい場合は、
GUI のエンコードセクションで**作業ディレクトリ**を設定するか、CLI フラグを
使ってください:

```bash
jasna --input input.mp4 --output output.mp4 --segments "10-25" --working-directory /fast/scratch
```
