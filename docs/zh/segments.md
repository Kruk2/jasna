# 只修复视频的一部分

你不必修复整个视频。只选需要修复的场景 — 视频的其余部分会原样复制，
这样处理速度快得多，其他部分也能保持原始质量。输出仍然是完整长度的
视频。

## 区间编辑器（GUI）

<img width="822" alt="Segment editor" src="https://github.com/user-attachments/assets/c67939a9-37de-46ae-b722-20c6a0933df7" />

点击任意队列视频上的剪刀按钮即可打开。

- 预览视频并按帧精确选择区间。不选择任何区间时会修复完整视频。
- **修复预览**可在正式处理前，用当前修复设置显示当前帧或短片段的
  效果。
- 选择了区间时，导出会保留源视频编解码器 — 主界面**编码设置**中的
  编解码器选项不会生效，因为未选择的部分会原样复制。如果某个视频
  无法使用区间处理，编辑器会提前告诉你。

### 马赛克扫描

编辑器可以帮你找出含马赛克的场景:

- 可逐帧扫描，或按 0.25–2 秒间隔扫描。扫描在 GPU 上运行，在
  **RTX 5090 上约为 2,000 FPS**。
- 扫描后调整置信度滑块，会立即更新琥珀色的检测区间，然后一键把它们
  加入紫色的修复选择。

检测模型和置信度会为队列中的每个视频单独记住，并用于最终处理，因此
不同视频可以使用不同设置。

### 提交更好的遮罩

当检测结果看起来不对时，你可以帮助改进未来的模型。暂停到该帧，点击
**提交更好的遮罩**，然后勾画每个马赛克区域 — 编辑器会引导你完成绘制。
如果马赛克边缘模糊或渐变，也请把这部分柔和区域包含进来。

提交时帧和遮罩会**匿名**上传。数据在你的电脑上加密后才上传，且仅
附带应用版本、检测模型名称和帧分辨率 — 绝不包含文件名、时间戳或任何
可识别你的信息。

## CLI: `--segments`

编辑器的 CLI 版本。时间可以用秒数或 `HH:MM:SS.s` 格式:

```bash
jasna --input input.mp4 --output output.mp4 --segments "10-25,01:10-01:30.5"
```

Jasna 会修复列出的区间并复制其余部分。区间附近最近的安全切点周围
会有一小段过渡被重新编码但不修复 — 这是把视频无缝拼接回去所必需的。

区间处理支持的情况:

- 目前仅支持 NVIDIA GPU。
- 一次一个视频（不支持文件夹、图像或流媒体）。
- 输入为恒定帧率的 H.264、HEVC 或 AV1；输出为 MP4、MOV 或 MKV。
- 输出编解码器始终与输入编解码器相同。
- 不能与 `--retarget-high-fps` 同时使用。

不兼容的输入会在处理开始前被拒绝，并给出清晰的错误提示。

## 工作目录

组装区间输出时，Jasna 会把临时文件存放在输出视频旁的隐藏文件夹中。
如果想把它们放到另一块（更快或更大的）磁盘上，请在 GUI 的编码设置
部分设置**工作目录**，或使用 CLI 参数:

```bash
jasna --input input.mp4 --output output.mp4 --segments "10-25" --working-directory /fast/scratch
```
